# Upgrading from ContextOS v1 → v2

> If you set up ContextOS Personal before 2026-04-14, your install is v1. v2 changes onboarding, de-InfoSec-ifies the core, and adds skills + hooks. This guide walks you through upgrading without losing your customizations.

**Last Updated:** 2026-04-14

---

## What Changed

### v1 (April 2026)
- Onboarding was prose in CLAUDE.md (no actual trigger — you had to know to run setup.py or hope Claude read the instructions)
- Core templates were InfoSec-flavored
- No skills directory
- No SessionStart hook
- `setup.py` was the reliable path

### v2 (2026-04-14)
- SessionStart hook deterministically triggers onboarding when `my-context.md` is missing
- Core templates are domain-neutral; InfoSec content moved to `examples/infosec/`
- New skills: `skills/onboarding.md`, `skills/prompt-framework.md`, `skills/chunking.md`
- Routing table pattern in CLAUDE.md
- `docs/common-mistakes.md` added

See [CHANGELOG.md](../CHANGELOG.md) for the full list.

---

## Your Personalized Files Are Safe

Your v1 `CLAUDE.md` and `my-context.md` (the ones generated by your onboarding) **don't need to change**. They still work. You just won't have the new routing table pattern unless you regenerate.

What you *will* want to add/update to get v2 benefits:

1. **Pull the new `skills/` directory** — unlocks `prompt-framework` and `chunking` without touching your personal files.
2. **Install the SessionStart hook** — useful if you ever delete `my-context.md` to re-onboard (or if a teammate clones your fork).
3. **Update `.claude/settings.json`** — adds the hook config.
4. **Rename `infosec-personal-os/` → `contextOS-personal/`** (if your local folder still has the old name).

## Upgrade Path

### Option 1: Cherry-pick the new files (recommended if you have local customizations)

```bash
cd ~/context-os
git fetch origin
git checkout origin/main -- skills/ docs/common-mistakes.md docs/upgrade-from-v1.md examples/README.md .claude/hooks/ .claude/commands/verify.md CHANGELOG.md
chmod +x .claude/hooks/check-first-run.sh
```

Then manually update `.claude/settings.json` to add the hook config. Merge with your existing MCP server / permissions config:

```json
{
  "permissions": {
    "deny": [
      "Edit(shared-context/**)",
      "Write(shared-context/**)",
      "Bash(rm shared-context/*)",
      "Bash(mv shared-context/*)",
      "Bash(cp * shared-context/*)"
    ]
  },
  "hooks": {
    "SessionStart": [
      {
        "matcher": "*",
        "hooks": [
          {
            "type": "command",
            "command": ".claude/hooks/check-first-run.sh"
          }
        ]
      }
    ]
  }
}
```

Don't touch your `CLAUDE.md` or `my-context.md`. They keep working.

### Option 2: Re-onboard from scratch

If your customizations are light (or you'd rather start fresh with the v2 structure):

```bash
# Back up your current context
cp CLAUDE.md CLAUDE.md.v1.bak
cp my-context.md my-context.md.v1.bak

# Pull v2 template
git fetch origin
git reset --hard origin/main

# Delete my-context.md to re-trigger onboarding
rm my-context.md

# Run Claude — SessionStart hook will detect first-run and launch the new 7-step flow
claude
```

Answer the questions again (pull from your `.v1.bak` files for reference). Takes ~10 minutes.

### Option 3: Merge manually

If you want the v2 CLAUDE.md structure (routing table pattern) but keep your answers:

1. Open the new [CLAUDE-TEMPLATE.md](../CLAUDE-TEMPLATE.md).
2. Copy-paste your v1 CLAUDE.md content into the template, matching sections.
3. Fill in the routing table with your tools and projects.
4. Save as your new `CLAUDE.md`.

## If Your Local Folder Name Is Old

If you cloned before 2026-04-14 when the repo was `infosec-personal-os`, your local folder may still be named that. Rename it:

```bash
cd ~
mv infosec-personal-os context-os
# Update any shell aliases or scripts that referenced the old path
```

The remote origin on GitHub was already renamed to `contextOS-personal`; no action needed there.

## Verification

After upgrading, run:

```bash
claude
```

Then ask: `/verify`

The verify command runs the new health check and reports any issues.

---

## Rollback

If something breaks, you can always restore your v1 state:

```bash
cp CLAUDE.md.v1.bak CLAUDE.md
cp my-context.md.v1.bak my-context.md
```

Open a GitHub issue on `contextOS-personal` if you hit anything unexpected.
